How It Works

Integrate in 3 simple steps

No SDK, no backend changes, no new database. SecureIDS sits invisibly in front of your existing login.

01

Configure Working Hours

Set your allowed business hours below. Any login attempt outside of your specified start and end time will be flagged.

02

Paste One Line of HTML

Copy the <script> tag with your API key and paste it anywhere on your login page. The widget auto-detects your login form.

03

You're Protected

Every login attempt is now silently screened for suspicious hours, failed attempts, and account status before your server even sees it.

Integration Code

Your integration snippet

Copy and paste this into the <body> of your login page. Replace YOUR_API_KEY with the key from the Admin Panel.

login.html 
<!-- ===== SecureIDS Invisible Shield ===== -->
<script src="{{ ids_host }}/static/ids-shield.js"
        data-api-key="{{ company.api_key }}"></script>
<!-- ======================================= -->

That's it. The widget automatically finds any <form> with a password field and intercepts it before submission.

API Reference

REST API Endpoints

For developers who want full control — call the API directly from your backend.

ENDPOINT 1 — Pre-screen (used by widget)
POST {{ ids_host }}/api/v1/auth/verify
Request Headers
HeaderRequiredDescription
AuthorizationYesBearer YOUR_API_KEY
Content-TypeYesapplication/json
Request Body
FieldTypeDescription
identifierstringUsername or email attempting login
urlstringThe page URL (optional, for logging)
Example Request
curl 
curl -X POST {{ ids_host }}/api/v1/auth/verify \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"identifier": "john.doe"}'
ENDPOINT 2 — Full Authentication (backend-to-backend)
POST {{ ids_host }}/api/v1/auth/login
Request Headers
HeaderRequiredDescription
AuthorizationYesBearer YOUR_API_KEY
Content-TypeYesapplication/json
Request Body
FieldTypeDescription
identifierstringUsername, email, or employee ID
passwordstringThe user's plain-text password
Example Request
curl 
curl -X POST {{ ids_host }}/api/v1/auth/login \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"identifier": "john", "password": "pass123"}'
Response Status Types
allow

Login Permitted

All security checks passed. Grant the user access to your system.

{
  "status": "allow",
  "reason": "Login successful.",
  "user": {
    "id": "abc-123",
    "name": "John Doe",
    "role": "Employee"
  }
}
deny

Login Denied

Failed credentials, outside work hours, or locked account. Refuse access.

{
  "status": "deny",
  "reason": "Incorrect credentials.
  2 attempt(s) remaining.",
  "failed_attempts": 1
}
lock_account

Account Locked

Max failed attempts reached. Account is now locked. Admin is notified by email.

{
  "status": "lock_account",
  "reason": "Account locked after
  3 failed attempts.",
  "failed_attempts": 3
}
